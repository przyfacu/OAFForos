# Diseño de OAFForos — MVP

## Propósito

OAFForos reúne dos experiencias complementarias para la comunidad argentina de olimpíadas de física: un foro abierto de discusión y un archivo curado de problemas. La lectura es pública; solo las cuentas verificadas pueden participar.

## Reglas de producto

- El foro no tiene votos, likes, reputación ni una respuesta “aceptada”. Las soluciones se revisan conversando.
- El archivo sigue siempre la ruta tipo de competencia → competencia → edición/año → nivel → problema.
- Cada problema publicado puede tener un único hilo asociado. Pistas y soluciones se marcan como spoiler y permanecen contraídas de entrada.
- Miembros proponen altas o correcciones al archivo; moderación decide publicarlas, pedir cambios o rechazarlas.
- Se admiten Markdown, LaTeX, imágenes y enlaces; no se almacenan PDFs en el MVP.

## Roles

| Rol | Capacidades |
| --- | --- |
| Visitante | Leer foro y archivo; buscar. |
| Miembro | Crear/editar sus temas y respuestas, reportar, proponer cambios de archivo. |
| Moderador | Revisar propuestas, reportes y ocultar/cerrar contenido. |
| Administrador | Todo lo anterior y asignar roles/configurar categorías. |

## Criterios de beta

La beta está lista cuando una persona puede registrarse con email o Google, crear una discusión, leer un problema del archivo, acceder a su hilo, proponer una entrada y cuando un moderador puede revisarla. Antes de abrir registro público se deben publicar pautas de convivencia, contacto de moderación y una política básica de privacidad.
